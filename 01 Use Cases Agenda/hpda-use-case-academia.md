### Use Case: University of Ontario – Verarbeitung von Sensordaten medizinischer Überwachungsgeräte

> source: (https://www.bitkom.org/sites/default/files/file/import/BITKOM-LF-big-data-2012-online1.pdf)[Bitkom]

![UOIT Logo](image-1.png)

- Verhinderung von Infektionen von Neugeborenen, die Tod des Kindes zur Folge haben können
- Infektionen sind vorhersagbar durch bestimmte Indikatoren
- Verschiedene Sensoren überwachen dauerhaft besagte Indikatoren, um frühzeitig Alarm schlagen zu können

Datenmerkmale: 
- Dateninput: Sensordaten aus verschiedenen medizinischen Messungen 
- Real Time Data, die sehr schnell verarbeitet werden muss
- Ziel: Warnung, wenn Indikatoren auf Infektion hinweisen. ML Klassifikatoren verwenden auf Daten 

Big Data Probleme: 
- Velocity -> Real Time Data muss enorm schnell ausgewertet werden, je früher Gefahr erkannt wird, desto höher die Chance den Infekt des Kindes zu verhindern
- Volume -> Verschiedenste Sensoren übermitteln jede Sekunde Daten 
- Variety -> unstrukturierte Daten, Daten von verschiedenen Sensoren/Quellen müssen kombiniert ausgewertet werden.


-> Abwehr lebensbedrohlicher Infekte durch Real Time Data Monitoring
